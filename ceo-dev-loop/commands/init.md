description: 현재 프로젝트에 CEO-Dev 루프 파일 구조를 설치합니다 (CLAUDE.md, docs/GOAL.md, docs/STATUS.md)
argument-hint: [프로젝트 목표 한 줄 요약]
---

사용자가 `/ceo-dev-loop:init` 명령을 실행했습니다. 아래 작업을 순서대로 수행하세요.

## 입력
- 프로젝트 목표: $ARGUMENTS (비어있으면 사용자에게 물어볼 것)

## 할 일

### 1. 기존 파일 확인
- 루트에 `CLAUDE.md`가 이미 있으면 → 내용 읽고, CEO-Dev 루프 섹션을 **추가**만 할 것 (기존 내용 유지)
- `docs/GOAL.md`, `docs/STATUS.md`가 이미 있으면 → 사용자에게 덮어쓸지 확인

### 2. `docs/` 폴더 생성 및 파일 작성

**`docs/GOAL.md`**
```markdown
# 프로젝트 목표

## 최종 목표
$ARGUMENTS

## 필수 요구사항
- [ ] (사용자에게 추가 작성 요청)

## 제약 조건
- 기술 스택:
- 사용 금지:

## 완료 기준 (DoD)
- [ ] 빌드/실행 성공
- [ ] 모든 필수 요구사항 구현
- [ ] 기본 동작 검증
```

**`docs/STATUS.md`**
```markdown
# 현재 상태

## 최근 작업
(시작 전)

## 현재 빌드/실행 상태
(시작 전)

## 다음 작업
CEO 지시 대기

## 알려진 이슈
없음
```

**`docs/DECISIONS.md`**
```markdown
# 결정 로그

CEO가 내린 주요 결정을 시간순으로 누적 기록합니다.
```

### 3. `CLAUDE.md`에 루프 규칙 추가 (없으면 새로 만들고, 있으면 아래 섹션만 append)

```markdown

## CEO-Dev 자동 루프 규칙

당신은 이 프로젝트의 메인 Dev 에이전트입니다. 아래 흐름을 엄격히 따르세요.

### 작업 흐름
1. `docs/GOAL.md`에서 목표 확인
2. 코드 작성 / 파일 생성
3. `docs/STATUS.md` 갱신 (완료 작업 / 현재 상태 / 결정 필요 사항)
4. `@ceo` 호출 → 검토 및 다음 지시 수신
5. 응답이 `[DONE]` 이면 종료, 아니면 CEO 지시에 따라 2번으로 반복

### @ceo 호출이 **반드시** 필요한 시점
- 기능 하나/파일 하나/리팩터 하나 단위 완료 시
- 라이브러리 선택, 아키텍처 분기, 트레이드오프 발생 시
- 요구사항이 모호하거나 방향이 불확실할 때
- 빌드 성공 / 테스트 통과 후 다음 단계 확인 시

### 호출 형식
```
@ceo 검토 요청

[완료한 작업]
- 생성/수정한 파일
- 핵심 변경 내용

[현재 상태]
- 빌드/실행 상태
- 알려진 이슈

[결정 필요 사항] (있을 때만)
- 옵션 A vs B
```

### 종료 조건
CEO 응답이 `[DONE]`으로 시작하면:
1. `docs/STATUS.md` 최종 상태 기록
2. 작업 중단
3. 사용자에게 최종 요약 보고

### 예외 (사용자 확인 필수)
- `git push`, 배포
- 외부 API 키/시크릿 필요 작업
- 파괴적 작업 (파일 삭제 대량, DB 마이그레이션)
그 외에는 사용자 확인 없이 CEO 승인으로 진행.
```

### 4. 완료 보고
생성한 파일 목록과 다음 단계 안내:
```
✅ CEO-Dev 루프 설치 완료

생성된 파일:
- CLAUDE.md (규칙 추가)
- docs/GOAL.md
- docs/STATUS.md
- docs/DECISIONS.md

다음 단계:
1. docs/GOAL.md 의 "필수 요구사항"을 구체적으로 채우세요
2. `/ceo-dev-loop:start` 로 루프를 시작하거나,
   그냥 "시작해" 라고 입력하면 자동으로 GOAL.md 기반 작업이 시작됩니다
```
